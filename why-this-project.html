Why I Built This Scrollytelling Test

This page is the branch from the homepage story. It explains the purpose, decisions, and what changed when I switched to spec-driven development.

Problem

AI can generate code quickly, but without constraints it produces drift: inconsistent styling, unclear structure, and fragile behavior.

Approach

I used a lightweight spec stack: write requirements, split into phases, implement in order, and verify each phase before moving on.

Result

The output became cleaner and easier to review. The homepage, content pages, and docs all align because they came from one process.

Step 1: Define the story arc.
Step 2: Specify page roles and links.
Step 3: Implement one phase at a time.
Step 4: Validate behavior and polish.
